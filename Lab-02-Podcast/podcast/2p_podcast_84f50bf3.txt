1) 准备与环境

2) 并行模型入门
Host: CUDA 的并行模型怎么理解？

Guest: 核心是大量轻量级线程的组织与协作：
- 线程（thread）是最小执行单元，按线程块（block）分组，线程块再组成网格（grid）。
- 每个线程有自己的寄存器和局部内存；块内线程可用共享内存（shared memory）协作，并用 __syncthreads() 同步。
- GPU 以 warp（通常 32 线程）为执行单位，warp 内的指令同时执行，分支会导致 warp 发散（divergence）。
- 编程要考虑索引映射：threadIdx、blockIdx、blockDim、gridDim。例如向量加中，每个线程负责一个元素，索引 = blockIdx.x * blockDim.x + threadIdx.x。

Host: 简单映射往往最有效，但良好索引设计是性能的第一步。

3) 内存与性能优化
Host: 性能的天花板常常在内存访问上。关键点有哪些？

Guest: 关注存储层级与访问模式：
- 存储层级：寄存器（最快但少）、共享内存（低延迟、块内共享）、全局内存（大但延迟高）、以及缓存（L1/L2）。
- 合并（coalesced）访问：保证连续线程访问连续地址，避免非对齐访问。
- 减少全局内存访问：把重用数据拉到 shared memory（如 tiled matrix multiply）。
- 避免共享内存 bank conflict：按访问模式组织数据或用 padding。
- 寄存器使用要平衡：寄存器占用太高会降低并发（occupancy）。
- 利用流（stream）重叠传输与计算：异步 memcpy + kernel 可以掩盖数据传输延迟。
- 选择合适的内存类型：常量内存或纹理内存对特定读模式有加速作用。

Host: occupancy（占用率）是常用指标，但不是越高越好，要结合寄存器、共享内存和块大小综合判断。

4) 内核设计与并行模式
Host: 写内核时有哪些常见技巧？

Guest: 一些实战经验：
- 尽量减少分支和避免 warp 发散，把分支移到块级或用 predication。
- 使用 warp 内原语（如 __shfl_sync）做快速归约或数据交换，能减少共享内存同步开销。
- 并行归约（reduction）：常用树形归约、分阶归约（先在 block 内归约，再跨 block 处理）。
- Tiled 算法：矩阵乘与卷积用 tiled + shared memory 能显著提升数据复用量。
- 复用高层库：在可用时优先用 cuBLAS、cuDNN、Thrust，避免重复造轮子。
- 写可配置的 kernel：把 blockDim/gridDim 参数化，便于后续的性能搜索（tuning）。

Host: 举例：矩阵乘从简单三层循环到 tiled + shared memory，往往能提升好几个数量级。

5) 工具与学习路径
Host: 想快速定位性能瓶颈，哪些工具和资源值得用？

- 系统级分析：Nsight Systems（查看 CPU/GPU 协同、stream、kernel 调度）。
- 内核级分析：Nsight Compute（每个 kernel 的寄存器、指令吞吐、内存带宽）。

- 测时技巧：用 cudaEventRecord 精确测时，先用小数据验证功能，再用大数据做性能分析。
- 学习路线：从 vector add、dot product、naive matrix multiply 开始；进阶做 tiled matrix multiply、卷积、并行归约、scan（prefix-sum）；参考 CUDA samples、Thrust 示例。
- 课程与书籍：Udacity、Coursera、《CUDA by Example》、《Programming Massively Parallel Processors》；社区有 NVIDIA Developer Forums、Stack Overflow、GitHub、r/CUDA 等。
